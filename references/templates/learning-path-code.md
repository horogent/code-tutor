## 学习节奏建议

| 每日时间 | 建议安排 |
|---------|---------|
| 15分钟 | 读一个知识点 + 跑通代码 |
| 30分钟 | 读一个知识点 + 跑通代码 + 做验证题 |
| 1小时 | 学 2-3 个知识点 + 动手实验 + 验证题 |
| 2小时+ | 学完一个模块 + 做模块综合验证 + 开始小项目 |

**黄金法则**：每天至少写 15 分钟代码。持续写 > 集中突击。
