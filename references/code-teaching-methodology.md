# 编程教学三层递进法

## 总览

编程教学与通用学科教学最大的区别在于：**编程有确定的、可观察的现象层**（代码运行结果）。

这使得编程教学可以实现"现象驱动"——永远可以先展示现象，再解释语义，最后归纳概念。这是编程教学得天独厚的优势。

核心教学路径（自下而上）：
```
现象（运行结果）→ 语义（代码含义）→ 概念（判别模型）→ 联结（命题模型）
```

---

## 一、三层递进详解

### 递进一：语义学习（现象驱动）

#### 1.1 目标
让学习者通过运行代码、观察输出来理解一个术语/语法/API"是干什么的"。

#### 1.2 教学步骤

**Step 1：给极简可运行代码**
```python
# 示例：教「列表推导式」
numbers = [1, 2, 3, 4, 5]
squares = [n * n for n in numbers]
print(squares)
```

**Step 2：展示运行结果（现象）**
```
[1, 4, 9, 16, 25]
```

**Step 3：引导现象观察**
> "你看到了什么？输入是 `[1,2,3,4,5]`，输出是 `[1,4,9,16,25]`。输入和输出的关系是什么？"

**Step 4：连接语义与现象**
> "你看到的这个一一对应的转换——把每个元素变成它的平方——就是「列表推导式」做的事。
> 用一句话说：列表推导式 = 在一行内对列表的每个元素做变换，生成新列表。"

**Step 5：变式验证（用未见情况）**
```python
# 新现象
words = ["hello", "world", "python"]
uppers = [w.upper() for w in words]
print(uppers)
```
```
['HELLO', 'WORLD', 'PYTHON']
```
> "现在你看到了：同一个语法结构，不同的变换操作。你能预测如果换成 `len(w)` 会输出什么吗？"

#### 1.3 语义学习的核心原则
- **一次一个语义**：每个语义学习单元只聚焦一个术语/语法
- **代码极简**：例子代码不超过10行，聚焦核心语义
- **现象突出**：输入和输出的对比要一目了然
- **变式验证**：换一个输入/操作，让学习者预测输出

### 递进二：概念构建（从现象归纳含义）

#### 2.1 目标
从多个语义实例中归纳出通用的判别规则，形成编程概念的判别模型。

#### 2.2 教学步骤

**Step A：激活现象层（先不给定义）**
展示 3 个正例代码 + 运行结果 + 2 个反例代码 + 运行结果

> 示例：教「高阶函数」概念
>
> 正例1：
> ```python
> def apply_twice(f, x):
>     return f(f(x))
>
> print(apply_twice(lambda n: n + 1, 5))  # 7
> ```
> 正例2：
> ```python
> numbers = [1, 2, 3]
> result = list(map(str, numbers))
> print(result)  # ['1', '2', '3']
> ```
> 正例3：
> ```python
> def make_adder(n):
>     return lambda x: x + n
>
> add3 = make_adder(3)
> print(add3(10))  # 13
> ```
>
> 反例1：
> ```python
> def add(a, b):
>     return a + b
> print(add(1, 2))  # 3 —— 接收的是数字，不是函数
> ```
> 反例2：
> ```python
> result = [1, 2, 3] * 2
> print(result)  # [1, 2, 3, 1, 2, 3] —— 操作列表，跟函数无关
> ```

**Step B：引导归纳共性**
> "这三个正例有什么共同的特点？"
> "反例跟它们有什么不同？"
>
> 引导方向：正例中的函数（`f`、`str`、`lambda`）被当作参数传递或作为返回值——函数被当作"数据"来操作。

**Step C：给出抽象描述（内涵）**
> "高阶函数的内涵（共有属性）：
> 1. 接收一个或多个函数作为参数，**或者**
> 2. 返回一个函数作为结果
>
> 判别规则：代码中是否有函数被当作参数传递或作为返回值？有 → 高阶函数。"

**Step D：正反例标注**
逐一标注每个正例反例的特征匹配情况。

**Step E：理解检查（必须用未见现象）**
提供一段新代码（用户从未见过），判断是否属于该概念。

| 能力 | 验证方式 |
|------|---------|
| 弱 | 选择题：3段新代码，哪段用到了高阶函数？ |
| 中 | 开放式：写一段用到高阶函数的代码 + 预测输出 |
| 强 | 边界挑战：`sorted(numbers, key=abs)`算不算高阶函数？`key`是函数吗？`abs`是函数吗？ |

> 对于弱/中基础：`sorted(numbers, key=abs)` 中的 `key=abs` 涉及"函数作为参数传递"——如果用户不知道 `key` 参数的含义，触发递进一（语义学习）。

**Step F：三层检查**
- 现象层："写一段用到高阶函数的新代码，运行，告诉我输出"
- 语义层："用自己的话说说，你的代码中哪部分体现了'函数被当作数据操作'"
- 概念层："如果不看代码运行结果，你怎么判断一段代码是不是高阶函数？"

### 递进三：知识构建（建立联结模型）

#### 3.1 目标
建立输入概念→输出概念的映射关系，使学习者能面对新需求时推导出正确的代码。

#### 3.2 教学步骤

**Step A：确认前置判别模型**
> "在建立这个映射之前，先确认：你能识别什么是[输入概念A]吗？什么是[输出概念B]吗？"
>
> 如果判别模型不牢 → 回到递进二（概念构建）。

**Step B：展示多个「输入→输出」对应实例**
以「递归解决问题」的联结模型为例：

| 输入（问题） | 输出（递归代码结构） |
|------------|------------------|
| 计算 n! | 基线条件: n≤1 → 1; 递归: n * factorial(n-1) |
| 计算斐波那契第 n 项 | 基线条件: n≤2 → 1; 递归: fib(n-1) + fib(n-2) |
| 反转字符串 | 基线条件: 字符串长度≤1 → 字符串本身; 递归: reverse(s[1:]) + s[0] |

**Step C：归纳通用映射**
> "从这三个例子中，你发现把一个「递归可解的问题」转换成「递归代码」的通用步骤是什么？"
>
> 归纳：
> 1. 识别基准条件（最简单、不需递归的情况）
> 2. 识别递归关系（如何把大问题分解为同类型的小问题）
> 3. 识别组合方式（如何把子问题的结果组合成最终结果）

**Step D：给出精确表述**
> 递归问题 → 递归代码的映射：
> 给定问题 P(n)：
> 1. 如果 n 已达到最小规模 → 返回基准值
> 2. 否则 → 计算 P(n-1)（或 P(n/2) 等）→ 用子结果组合成当前结果

**Step E：未见验证**
> "现在给你一个没用过的输入：'遍历二叉树的所有节点并求和'。用刚才的映射模型，写出递归代码。"

**Step F：适用范围讨论**
> "什么情况下递归不适用？如果数据量太大导致栈溢出怎么办？尾递归优化是什么？"

## 二、综合编程教学示例：教「Promise/异步」

### 语义依赖检测
教 Promise，需要先检查：
- 用户知道什么是"回调函数"吗？→ 未知 → 触发递进一
- 用户知道什么是"异步"吗？→ 未知 → 触发递进一
- 用户知道 `then`/`catch` 语法吗？→ 未知 → 正常教学

### 教学顺序（按依赖）
1. 递进一：异步 vs 同步（用 `setTimeout` 产生现象）
2. 递进一：回调函数（用事件监听产生现象）
3. 递进二：Promise 概念构建（正例：链式调用；反例：回调地狱）
4. 递进三：需求→Promise 代码的联结模型

## 三、误差教学法（编程特有）

利用错误输出（bug 的现象）作为教学手段。

### 原理
错误输出是极其丰富的现象层材料——它展示了"边界条件""类型不匹配""逻辑错误"等各种概念的**反例**。

### 方法
1. 故意展示有 bug 的代码
2. 展示报错信息/错误输出（现象）
3. 引导分析：为什么会产生这个错误？（语义）
4. 归纳：这类错误属于什么类型？（概念——错误类型判别模型）
5. 联结：以后遇到这类报错→应该检查什么？（联结模型）

### 示例
```python
# bug 代码
def get_average(numbers):
    return sum(numbers) / len(numbers)

print(get_average([]))
```
```
# 现象（报错）
ZeroDivisionError: division by zero
```
> 语义分析："`len([])` 返回 0，`sum([])` 返回 0，`0/0` 虽然数学上没意义但 Python 中 `0/0` 是..."
> 等等，我们先运行确认 → 其实是 `ZeroDivisionError`，因为分母为 0
>
> 概念归纳：这是「边界条件未处理」类错误 + 「空集合」边界情况
>
> 联结模型：凡涉及除法/取平均/比例 → 检查分母是否可能为 0 → 提前处理边界
